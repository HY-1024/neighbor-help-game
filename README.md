# 《邻里帮忙》暑期寻物游戏说明文档

## 1. 作品名称

作品名称：《邻里帮忙》

副标题：暑期寻物

作品类型：基于 HTML / JavaScript / Three.js 的网页寻物游戏

## 2. 游戏主题说明

本作品的主题是“老旧小区里的邻里互助”。

故事发生在暑假早晨，玩家住在老旧小区 5 号楼。楼下王奶奶买菜回来发现钥匙不见了，三楼的小朋友丢了篮球，门卫张大爷的老花镜也找不到了。玩家需要在小区中观察环境，帮助邻居找回丢失的物品，最终获得“社区小帮手”的称号。

游戏试图表现老旧小区的生活感和烟火气：红砖居民楼、绿藤蔓、旧式路灯、自行车棚、花坛、石桌石凳、健身器材、晾衣绳、空调外机等元素共同组成了寻物空间。

## 3. 目标和基本玩法

游戏目标：

- 在限定时间内找到清单中的全部物品
- 点击正确物品获得分数
- 尽量减少错误点击
- 在倒计时结束前完成寻物任务

基本玩法：

- 玩家进入游戏后选择二维模式或三维模式
- 选择简单、中等或困难难度
- 根据左侧清单寻找场景中的物品
- 点击正确物品后，该物品消失并计分
- 点击错误位置会扣分
- 多次找错后，系统会逐步给出更详细提示
- 找齐全部物品后显示胜利页面
- 时间归零后显示失败页面

## 4. 难度与参数规则

| 难度 | 需要寻找的物品数量 | 时间限制 |
| --- | ---: | ---: |
| 简单 | 3 件 | 120 秒 |
| 中等 | 5 件 | 90 秒 |
| 困难 | 8 件 | 60 秒 |

得分规则：

- 点击正确物品：+20 分
- 点击错误位置：-2 分
- 分数最低不低于 0

提示规则：

- 连续错误点击达到一定次数后，清单提示会变得更具体
- 第一次阶段提示只给出大致方向
- 第二次阶段提示给出详细位置

## 5. 空间生成逻辑说明

游戏空间由“核心小区区域”和“外围延伸街区”两部分构成。

核心小区区域包括：

- 5 层红砖居民楼
- 楼前水泥空地
- 花坛
- 自行车棚
- 石桌石凳
- 健身器材
- 长椅
- 老式路灯
- 公告栏

外围延伸街区包括：

- 外圈街道
- 人行道
- 路沿
- 斑马线
- 树带
- 路灯
- 周边低矮建筑

空间生成方式：

- 使用 Three.js 创建基础场景、相机、灯光和地面
- 使用 BoxGeometry 搭建楼体、道路、台阶、车棚、长椅、公告栏等方块结构
- 使用 CylinderGeometry 搭建路灯、桌椅、健身器材、自行车车轮、杆件等圆柱结构
- 使用 TorusGeometry 搭建自行车轮胎、健身器材圆环等环形物体
- 使用 Sprite 显示需要寻找的真实物品图标
- 使用 Raycaster 检测玩家是否点击到正确物品

## 6. 网格规则

虽然游戏画面不是传统棋盘，但场景布局参考了网格化设计思想。

主要网格规则：

- 地面被划分为规则的水泥砖块，方便玩家观察方向和距离
- 主要建筑沿 X / Z 方向排列，形成稳定的轴测空间
- 花坛、石桌、车棚、健身区分别占据不同区域，避免功能重叠
- 道具位置被放置在对应功能区附近，增强逻辑性
- 外圈街道围绕核心区域形成边界，减少场景突然结束的感觉

## 7. 邻接关系规则

本游戏中的空间不是随机堆放，而是按照现实小区中的邻接关系组织。

邻接关系示例：

- 居民楼门口邻接台阶和公告栏
- 台阶附近可以放置馒头、报纸卷等生活物品
- 花坛邻接楼前空地，适合放置钥匙等小物件
- 自行车棚邻接道路和空地，篮球可能滚到车棚角落
- 石桌石凳位于休闲区，老花镜放在石桌上符合生活逻辑
- 健身器材位于活动区，学生卡挂在扶手上符合寻物主题
- 外圈街道邻接小区边界，使地图看起来可以继续延伸

这些邻接关系让玩家能够通过生活经验推测物品位置，而不是完全随机寻找。

## 8. 随机规则

游戏中的核心寻物目标位置是固定设计的，保证作业演示时体验稳定。

同时，部分视觉细节采用轻微随机：

- 地面裂纹位置随机
- 绿植叶片位置随机
- 墙面藤蔓位置随机
- 小草、碎叶等装饰位置随机

这样可以让场景不显得完全机械，同时不会影响主要玩法。

## 9. 路径规则

游戏中的路径规则主要体现在空间动线和视线动线上。

玩家观察路径：

- 从居民楼正面开始观察
- 向花坛、石桌、车棚、健身区逐步移动视线
- 在三维模式中通过旋转和缩放查看被遮挡的区域

场景路径：

- 楼前空地是主要活动中心
- 外圈街道形成环形边界
- 人行道连接居民楼、车棚、花坛和休闲区
- 路灯和树带强化街道延伸感

三维模式下，玩家的路径不是角色移动，而是通过相机旋转和缩放完成空间探索。

## 10. 功能分区规则

游戏场景分为以下功能区：

| 功能区 | 场景内容 | 对应物品 |
| --- | --- | --- |
| 居民楼入口区 | 门口、台阶、公告栏 | 馒头、报纸卷 |
| 花坛区 | 花坛、泥土、绿植 | 钥匙 |
| 车棚区 | 自行车棚、自行车 | 篮球 |
| 休闲区 | 石桌、石凳、长椅 | 老花镜、玩具车 |
| 健身区 | 健身器材、扶手 | 学生卡 |
| 街道延伸区 | 外圈道路、树、路灯 | 增强空间完整性 |

功能区的划分让物品位置更加合理，也能帮助玩家形成寻找策略。

## 11. 物品清单与位置设计

基础物品：

- 钥匙：花坛边的泥土上
- 篮球：自行车棚角落的地面
- 老花镜：石桌上下棋处
- 学生卡：健身器材扶手上
- 一袋馒头：居民楼门口台阶上

困难模式新增剧情物品：

- 红伞：靠近路灯或墙边
- 玩具车：长椅附近的地面
- 报纸卷：公告栏或楼门口附近

物品摆放原则：

- 物品必须真实可见
- 物品不能只用文字标签代替
- 物品位置要符合现实生活逻辑
- 点击检测区域要与物品位置基本一致

## 12. 代码结构解释

本项目主要代码都写在 `neighbor-help.html` 中，便于提交和运行。

主要结构如下：

### 12.1 HTML 页面结构

负责页面基础内容：

- 启动页面
- 游戏 UI 面板
- 物品清单
- 倒计时和得分区域
- 声音控制区域
- 小地图
- 结算弹窗

### 12.2 CSS 界面绘制

负责视觉样式：

- 页面布局
- 面板样式
- 启动页背景
- 按钮样式
- 清单样式
- 结算页面样式
- 道具图标动画
- 响应式适配

### 12.3 Three.js 空间生成

相关函数包括：

- `setup3d()`：初始化 Three.js 场景、相机、渲染器和控制器
- `buildScene()`：组织整个三维小区场景
- `buildBuilding()`：生成红砖居民楼
- `buildRoadExtension()`：生成外围街道和延伸区域
- `buildOuterStreet()`：生成外圈街道、路沿、树带和路灯
- `buildBikeShed()`：生成自行车棚
- `addBike()`：生成自行车模型
- `buildFlowerBed()`：生成花坛
- `buildStoneTable()`：生成石桌石凳
- `buildOutdoorGym()`：生成健身器材

### 12.4 玩家交互

主要包括：

- 二维地图拖动
- 二维地图缩放
- 三维场景旋转
- 三维场景缩放
- 点击物品
- 点击错误位置
- 重置视图
- 退出游戏

相关函数：

- `onPointerDown()`
- `onPointerUp()`
- `onPointerMove()`
- `onMapWheel()`
- `onMapPointerDown()`
- `onMapPointerMove()`
- `onMapPointerUp()`
- `resetView()`
- `exitGame()`

### 12.5 碰撞检测 / 点击检测

三维点击检测使用 Three.js 的 `Raycaster`。

相关函数：

- `get3dHit()`
- `pick3d()`

基本逻辑：

- 将鼠标点击位置转换为标准屏幕坐标
- 从相机向场景发射射线
- 判断射线是否与可点击物品相交
- 如果点击到正确物品，则执行拾取逻辑
- 如果没有点击到物品，则执行扣分逻辑

### 12.6 得分机制

相关函数：

- `collect()`：正确拾取物品
- `miss()`：错误点击
- `updateProgress()`：更新进度
- `endGame()`：结束游戏并显示结算界面

得分逻辑：

- 正确拾取：增加 20 分
- 错误点击：扣除 2 分
- 找齐物品：胜利
- 时间耗尽：失败

### 12.7 参数控制

参数控制包括：

- 难度选择
- 模式选择
- 倒计时
- 得分
- 物品数量
- 音效开关
- BGM 开关
- 音量滑条

相关函数：

- `selectView()`
- `selectLevel()`
- `prepareRound()`
- `syncAudioUi()`
- `startBgm()`
- `stopBgm()`
- `applyBgmVolume()`

## 13. AI 编程过程记录

本项目在制作过程中使用 AI 辅助完成了主题构思、代码生成、三维场景搭建和错误修改。

### 13.1 向 AI 提出的需求示例

最初提出的需求包括：

- 制作一个老旧小区主题的寻物游戏
- 使用 HTML、JavaScript 和 Three.js
- 场景采用轴测或等距视角
- 玩家点击物品进行拾取
- 有倒计时、得分、胜利和失败页面
- 所有代码尽量放在一个 HTML 文件中

后续继续向 AI 提出修改需求：

- 加入启动页面
- 增加简单、中等、困难三种难度
- 支持二维和三维两种模式
- 三维场景要支持旋转和缩放
- 物品要真实可见，不能只是文字提示
- 增加 BGM 和音效开关
- 优化自行车、石桌、健身器材等模型
- 修复穿模问题
- 增加外圈街道，减少地图突然结束的问题
- 增加退出游戏按钮和结算页面

### 13.2 AI 如何帮助逐步完善功能

AI 的帮助主要体现在：

- 根据主题生成游戏规则和物品清单
- 生成 HTML / CSS / JavaScript 基础结构
- 使用 Three.js 搭建居民楼、道路和小区设施
- 添加二维地图和三维模式切换
- 添加计时、得分、扣分和结算逻辑
- 添加音效、BGM、音量控制
- 根据截图反馈修改模型穿模和视觉问题
- 根据课程要求整理 README 文档

### 13.3 如何修改错误

开发过程中出现过一些问题，例如：

- 自行车模型抽象，轮胎和车架不协调
- 部分模型出现穿模或悬浮
- 建筑背面过于空白
- 周边房屋在低角度观察时透视感不自然
- 二维页面中 BGM 关闭不彻底
- 结算页面功能不完整

修改方式：

- 重新调整自行车轮胎和车架的三维方向
- 把靠墙物体向外移动，减少穿墙
- 给建筑背面增加窗户、空调、管线和绿植
- 在外围增加街道、路沿、树带和路灯，弱化边缘透视问题
- 给 BGM 加入播放请求编号，避免异步加载后继续播放
- 增加退出游戏和返回首页逻辑

## 14. 本地运行方法

不要直接双击 `neighbor-help.html` 打开，推荐使用本地服务器。

### Windows 运行步骤

1. 打开项目文件夹：

2. 在文件夹空白处按住 `Shift`，点击鼠标右键。

3. 选择“在终端中打开”或“在此处打开 PowerShell 窗口”。

4. 输入：

   ```powershell
   python -m http.server 8080
   ```

5. 打开浏览器访问：

   ```text
   http://127.0.0.1:8080/neighbor-help.html
   ```

如果 `python` 命令不可用，可以尝试：

```powershell
py -m http.server 8080
```

如果 8080 端口被占用，可以换成：

```powershell
python -m http.server 8081
```

然后访问：

```text
http://127.0.0.1:8081/neighbor-help.html
```

## 19. 反思总结

通过本项目，我理解了一个网页游戏不仅需要实现基本功能，还需要考虑空间逻辑、视觉完整性和玩家体验。

AI 在编程中的帮助主要包括：

- 快速生成游戏原型
- 帮助组织代码结构
- 根据需求补充功能
- 根据截图反馈定位问题
- 辅助修改模型穿模、交互失效、音效控制等问题
- 帮助整理课程提交文档

同时也发现，AI 生成的代码并不是一次就完美的。三维建模中容易出现穿模、比例不合理、视角不自然等问题，需要通过不断测试、截图观察和修改参数来完善。最终作品是在多次反馈和修改后逐步完成的。

本项目的收获是：AI 可以提高开发效率，但最终效果仍然需要创作者不断判断、筛选、调试和完善。
